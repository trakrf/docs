# TRA-598 — BB17 W1 + W3 — Quickstart browser-only mint callout, canonicalize spec URLs

## Goal

Two docs-side fixes, single PR:

- **W1.** Quickstart needs an explicit acknowledgment that minting an API key is a browser-only flow, with a sentence telling CI/headless setups what to do instead (mint out-of-band, paste into a secret store, reference by env var).
- **W3.** Pick one canonical URL for the OpenAPI spec and reference it consistently across docs. Redirect non-canonical paths so existing links don't rot.

No platform or spec-content changes. Prose + redirect config + BB.md update.

## Background

### W1

Per `/docs/api/authentication`, the only path to mint an API key is the SPA (avatar menu → API Keys → New Key). The Quickstart's step 2 currently hand-waves this — a backend-engineer-with-curl who follows the Quickstart cold cannot complete it without a browser session, and will hunt for an HTTP path that doesn't exist before discovering the constraint.

The policy is correct (TRA-590 reframed Authentication around SPA-first onboarding). The fix is just a callout.

### W3

Three URLs serve the spec, all 200, semantically identical:

- `$DOCS_URL/redocusaurus/trakrf-api.yaml` — the path BB.md uses.
- `$DOCS_URL/api/openapi.yaml` — what Quickstart + Postman already point at.
- `$DOCS_URL/api/openapi.json` — JSON variant of the above.

Plus a cache-busted `assets/files/openapi-<hash>.yaml` surfaced as a `<link>` from the redoc page; this is an internal redoc artifact, not user-facing copy.

Three docs surfaces, no canonical choice. Third re-raise (BB14 → BB15 O-2 → BB16 W7 → BB17 W3). Pick one and stop relitigating.

## Approach

### W1 — Quickstart callout

Add a Docusaurus `:::note` admonition immediately after the existing step 2 mint paragraph (`docs/api/quickstart.mdx`, after the paragraph ending "[Mint your first API key](./authentication#mint-your-first-api-key)."). Content:

> :::note Browser required to mint a key
> Minting an API key requires an interactive browser session — there is no programmatic mint endpoint by design. CI/headless setups should mint a key out-of-band, paste it into a secret store (env var, vault, GitHub Actions secret), and reference it from there. See [Authentication → Where keys come from](./authentication#where-keys-come-from) for the design rationale and a contact path if your use case genuinely needs programmatic provisioning.
> :::

The Quickstart's own intro already points readers at the mint flow ("If you don't have a key yet, mint one from the SPA's avatar menu …") — leave that intact, the callout reinforces inside the actual step where someone mid-task would notice.

### W3 — Canonical URL + redirect

**Canonical:** `/api/openapi.json` and `/api/openapi.yaml`. Already public, already what Quickstart + Postman reference, doesn't leak Docusaurus internals.

**Redirect strategy:** Cloudflare Pages `_redirects` file in `static/`. Cloudflare Pages reads `static/_redirects` natively at deploy time, applies edge-level 301s. Doesn't require a Docusaurus plugin, doesn't ship JS shims, doesn't depend on client-side script.

Create `static/_redirects` with:

```
/redocusaurus/trakrf-api.yaml  /api/openapi.yaml  301
```

The cache-busted `assets/files/openapi-<hash>.yaml` URL is generated by redoc's link rendering on the `/api` page. It's not in any prose doc and not in BB.md — leave it untouched. It's an internal redoc artifact; users who copy that URL out of the rendered HTML are off the supported path.

**BB.md update:** `tests/blackbox/BB.md` line 51 references `$API_TEST_DOCS_URL/redocusaurus/trakrf-api.yaml`. Update to `$API_TEST_DOCS_URL/api/openapi.yaml`. The redirect ensures any older copy of BB.md still works, but BB.md is the prompt that future cycle-runners read first; the canonical path belongs in the prompt.

### Adjacent-surface audit

Per ticket, run two grep sweeps and document findings in a Linear comment.

1. **Residual non-canonical refs.** After the redirect lands, grep:
   - `redocusaurus/` across `docs/`, `static/`, `src/`, `tests/`, `*.md`, `*.ts`, `*.mdx`
   - `assets/files/openapi-` across the same surfaces
   Expect zero hits in `docs/` after the BB.md edit. Hits inside `node_modules/`, build output, or the `_redirects` file itself are fine.

2. **SPA-only steps that hand-wave.** Scan Quickstart and Authentication for other actions that silently require the browser: key revocation, scope changes, key rotation, organization switching. The Authentication page already documents the lifecycle (`Key lifecycle` section) but doesn't explicitly say "all of these are SPA-only." Worth a small umbrella callout if the audit confirms the pattern.

Document the audit (where I looked, what I found, what I didn't) as a comment on TRA-598. File pre-launch findings as new tickets under TRA-596; append post-launch findings to TRA-597.
