## What this page does

Locations lets you model the physical places assets live — buildings, rooms, shelves, vehicles — as a hierarchical tree. The left column lists or searches locations and the right column shows details for the selected one; on a fresh account both are empty with "No locations found. Create a root location to get started." and a floating **+** button. Footer cards summarize **Total Locations**, **Active**, **Inactive**, and **Root Locations** (top-level entries).

## How it fits in the app

Locations give **Assets** a home: each asset can be assigned to one, and **Inventory** can stamp scans with the current location (picked up from a location-marker tag, shown on this site as "No location tag detected"). **Reports** uses these locations to produce movement history and "last seen" summaries.
